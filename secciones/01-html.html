HTML y el punto de entrada

Cómo el navegador empieza a ejecutar el proyecto: index.html y main.jsx.

Toda página web empieza con un fichero HTML. En proyectos React ese fichero es mínimo: sólo contiene una caja vacía y un script. React se encarga del resto. En esta sección lo desmenuzamos línea a línea para que sepas qué pasa exactamente cuando alguien escribe la URL en el navegador.

1. ¿Qué es HTML?

HTML (HyperText Markup Language) es un lenguaje de etiquetas. Cada etiqueta representa un trozo de la página: títulos, párrafos, imágenes, formularios… El navegador lee el HTML de arriba abajo y construye un árbol en memoria llamado DOM (Document Object Model).

Idea clave

El DOM es como un árbol genealógico de tu página: html es el abuelo, body es un hijo, dentro hay div, dentro hay p… JavaScript puede manipular ese árbol en vivo: añadir hijos, cambiar texto, etc. Eso es lo que hace React por debajo.

Ejemplo mínimo de HTML

ejemplo.html 
<!doctype html>
<html lang="es">
  <head>
    <title>Hola</title>
  </head>
  <body>
    <h1>Hola, mundo</h1>
    <p>Bienvenido a la web.</p>
  </body>
</html>

2. El index.html de DaWeb

Está en daweb/daweb/index.html. Aquí lo tienes íntegro:

daweb/index.html 
<!doctype html>
<html lang="es">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <link rel="icon" type="image/svg+xml" href="/favicon.svg" />
    <link rel="preconnect" href="https://fonts.googleapis.com" />
    <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
    <link href="https://fonts.googleapis.com/css2?family=Google+Sans+Flex:opsz,wght@6..144,1..1000&display=swap" rel="stylesheet" />
    <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/bootstrap-icons@1.11.3/font/bootstrap-icons.min.css" />
    <title>DaWeb Reventas</title>
  </head>
  <body>
    <div id="root"></div>
    <script type="module" src="/src/main.jsx"></script>
  </body>
</html>

Línea a línea

LíneaQué hace
<!doctype html>Le dice al navegador "esto es HTML5". Si lo omites, entra en modo "quirks" (raro) que comparte raros bugs antiguos.
<html lang="es">Idioma de la página. Ayuda a buscadores, lectores de pantalla y al corrector ortográfico.
<meta charset="utf-8">Codificación: UTF-8 soporta tildes, eñes, emojis, todo.
<meta name="viewport" ...>Imprescindible para móvil: dice que el ancho de la pantalla es el ancho del dispositivo (sin esto el móvil "haría zoom out" automáticamente).
<link rel="icon" ...>Favicon: el iconito de la pestaña.
<link rel="preconnect" ...>Pista al navegador para abrir conexión con Google Fonts cuanto antes (ahorra unos ms en carga).
<link href=".../family=Google+Sans+Flex..." />Carga la fuente Google Sans Flex desde el CDN de Google.
<link rel="stylesheet" href=".../bootstrap-icons..." />Carga la fuente de iconos Bootstrap Icons (esos <i class="bi bi-..."> que verás).
<title>DaWeb Reventas</title>Texto de la pestaña del navegador.
<div id="root"></div>⭐ La caja donde React inyectará toda la web.
<script type="module" src="/src/main.jsx">Carga el JavaScript de entrada. type="module" activa los ES Modules, que permiten usar import / export.
La idea clave

En toda app React, el HTML es así de minúsculo: una div vacía con id root y un script. Toda la interfaz se construye después con JavaScript. Esto se llama SPA (Single Page Application): una sola página HTML que cambia su contenido sin recargar.

3. El arranque de React: src/main.jsx

src/main.jsx 
import { StrictMode } from 'react'
import { createRoot } from 'react-dom/client'
import 'bootstrap/dist/css/bootstrap.min.css'
import './index.css'
import './theme.css'
import App from './App.jsx'

createRoot(document.getElementById('root')).render(
  <StrictMode>
    <App />
  </StrictMode>,
)

Línea a línea

  • import { StrictMode } from 'react': trae el componente StrictMode que avisa de prácticas obsoletas durante el desarrollo.
  • import { createRoot } from 'react-dom/client': react-dom es el puente entre React y el DOM real del navegador. createRoot es el API moderno (React 18+) para iniciar una app.
  • import 'bootstrap/...';: importa el CSS de Bootstrap como efecto secundario (no necesita variable).
  • import './index.css' y './theme.css': cargan los estilos globales y el sistema de colores del proyecto.
  • import App from './App.jsx': trae el componente raíz.
  • createRoot(document.getElementById('root')).render(...): "toma la div con id root y dibuja dentro este árbol de componentes".
  • <StrictMode><App /></StrictMode>: envuelve la app en StrictMode (puro consejo: detecta efectos secundarios mal hechos).

Flujo completo desde la URL

1 Usuario escribe http://localhost:5173/.
▼
2 Vite responde con index.html.
▼
3 El navegador lee la etiqueta <script> y descarga /src/main.jsx (Vite lo compila al vuelo).
▼
4 main.jsx ejecuta createRoot(...).render(<App />).
▼
5 React monta el componente App dentro del div#root. ¡La página aparece!

4. ¿Qué son las "etiquetas con clases" como <i class="bi bi-search">?

El paquete Bootstrap Icons funciona inyectando una fuente: cada clase bi-XXX renderiza un carácter especial que dibuja un icono. Por eso usas <i> (italic) vacío con la clase apropiada.

5. Teoría profunda: lo que el entrevistador sabe

El pipeline de renderizado del navegador paso a paso

Cuando el navegador recibe el HTML del servidor, no lo muestra directamente. Pasa por un pipeline complejo:

1 HTML parsing: el parser lee el HTML de arriba abajo y construye el árbol DOM (Document Object Model) — una estructura de objetos JavaScript en memoria. Cada etiqueta es un nodo.
▼
2 Recursos bloqueantes: si el parser encuentra un <script> sin defer/async, detiene el parsing, descarga el script y lo ejecuta. Por eso los scripts sin defer deben ir al final del body o usar defer.
▼
3 CSS parsing: el CSSOM (CSS Object Model) se construye en paralelo con el DOM.
▼
4 Render Tree: DOM + CSSOM = árbol de lo que realmente se dibuja (excluye display:none, <head>, etc.).
▼
5 Layout: calcula posición y tamaño de cada elemento.
▼
6 Paint: dibuja píxeles en pantalla.

Por qué type="module" es especial

Los módulos ES tienen comportamiento distinto a los scripts clásicos:

CaracterísticaScript clásicotype="module"
Bloquea el parserSí (si está en head sin defer)No — siempre diferido
Modo estrictoOpcional ('use strict')Siempre activado
import/exportSyntaxErrorFunciona
ScopeGlobal — las variables son window.xLocal al módulo
Se ejecutaInmediatamente al descargarseCuando el DOM está listo (como DOMContentLoaded)
Por qué el script puede estar en el <head>

En HTML clásico, poner un script en el <head> sin defer bloquea el renderizado. Con type="module" es seguro en el <head> porque el módulo se descarga en paralelo y sólo ejecuta cuando el DOM está construido.

El atributo crossorigin en preconnect

El index.html tiene:

<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />

El atributo crossorigin le dice al navegador que la futura petición a ese servidor necesitará credenciales CORS anónimas. Sin él, el navegador establece dos conexiones diferentes (una con credenciales, otra sin) y el preconnect no sirve de nada para las fuentes. Con él, reutiliza la conexión preestablecida.

StrictMode: por qué tu efecto corre dos veces en desarrollo

En producción, cuando un componente se monta, React ejecuta el useEffect una sola vez. En desarrollo con StrictMode, React hace deliberadamente:

1 Monta el componente y ejecuta los efectos.
▼
2 Desmonta el componente y ejecuta las funciones de limpieza.
▼
3 Monta de nuevo y ejecuta los efectos otra vez.

El objetivo: detectar efectos sin función de limpieza correcta. Si tu efecto suma una visualización y no cancela el intervalo o subscription correctamente, lo verás roto en desarrollo antes de llegar a producción. Por eso DetalleProducto usa visualizacionContadaRef: el ref recuerda qué producto ya contó y no cuenta dos veces aunque el efecto corra dos veces.

SPA: la trampa del refresh en producción

En npm run dev, si abres directamente http://localhost:5173/productos funciona. Vite está configurado para servir index.html para cualquier ruta desconocida. En producción (nginx por ejemplo), al navegar a /productos, el servidor busca el fichero físico /productos, no lo encuentra y devuelve 404. Solución estándar: configurar try_files $uri /index.html; en nginx para que cualquier ruta sirva el index.html y React Router decida qué mostrar.

Preguntas trampa del entrevistador

"¿Qué pasa si el usuario tiene JavaScript desactivado?"

Respuesta completa: La web muestra un <div id="root"> vacío. No hay fallback de contenido porque es una SPA pura (Client-Side Rendering). Para manejar esto se puede añadir un <noscript> en el HTML, o usar SSR (Server-Side Rendering) con frameworks como Next.js, que genera el HTML en el servidor. DaWeb no usa SSR.

"¿Afecta esta arquitectura al SEO?"

Respuesta completa: Sí, negativamente. Los buscadores como Googlebot pueden ejecutar JavaScript, pero hay limitaciones: el crawler puede no esperar suficiente tiempo para que React monte los componentes, y puede que no ejecute ciertos scripts. Una SPA pura tiene peor indexación que HTML estático. Para un marketplace de segunda mano en producción real, se usaría SSR o pre-rendering para las páginas de producto.

"¿Por qué hay dos <link rel="preconnect"> para Google Fonts?"

Respuesta: Las fuentes de Google usan dos dominios: fonts.googleapis.com para la CSS que lista las fuentes (petición sencilla), y fonts.gstatic.com para los archivos de fuente reales (petición con CORS anónimo). El preconnect a ambos establece las conexiones TCP+TLS antes de que el navegador sepa que las necesita, ahorrando 100-200ms en la carga inicial.

5. Ejercicios

Ejercicio 1 Cambiar el título de la pestaña ★ Muy fácil
  1. Abre daweb/index.html.
  2. Cambia <title>DaWeb Reventas</title> por <title>Marketplace de prueba</title>.
  3. Guarda y mira la pestaña del navegador.
Si no se actualiza

Vite recarga el HTML completo al modificarlo (no es HMR, recarga la pestaña). Si no aparece el nuevo título, recarga manualmente con Ctrl+R.

Ejercicio 2 Añadir una fuente de Google adicional ★★ Intermedio
  1. En index.html, añade después del <link> de Google Sans Flex otra fuente: <link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;700&display=swap" rel="stylesheet" />
  2. En src/theme.css, cambia --sans (si lo encuentras) o añade en body: font-family: 'Inter', sans-serif; y observa el cambio.
Pista

Las fuentes se cargan en HTML pero se aplican desde CSS con font-family: 'Nombre', fallback;.

Ejercicio 3 Romper React a propósito ★★ Intermedio
  1. En index.html, cambia <div id="root"></div> por <div id="raiz"></div>.
  2. Guarda. La página estará en blanco.
  3. Abre la consola (F12 → Console) y verás un error tipo Cannot read properties of null (reading ...') porque document.getElementById('root') devuelve null.
  4. Deshaz el cambio.
¿Qué hemos aprendido?

El id del div en index.html tiene que coincidir exactamente con el getElementById en main.jsx. Es el único acoplamiento entre HTML y JS.

Ejercicio 4 Crear contenido "no-React" antes de que cargue ★★★ Avanzado

Pon dentro del <div id="root"> un texto: <p>Cargando…</p>. Recarga la página. ¿Qué ves? ¿Por qué desaparece?

Respuesta

Se ve "Cargando…" durante una fracción de segundo y desaparece. React, al renderizar, sustituye todo el contenido del #root por su propio árbol. Esta técnica se usa como splash screen para mostrar algo mientras el JS carga.
