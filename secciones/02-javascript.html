JavaScript moderno (ES6+)

Las piezas del lenguaje que sostienen todo el código de DaWeb, en el orden en que conviene aprenderlas.

Casi cualquier línea del proyecto usa una decena de "trucos" introducidos a partir de 2015 (ES6+): destructuring, arrow functions, spread, async/await… Si no los dominas, no entenderás luego ni React ni el cliente HTTP. En esta sección las recorremos en el orden en que se construyen unas sobre otras, con el ejemplo del proyecto pegado a cada concepto.

El plan

Variables → arrow functions → template literals → destructuring → spread/rest → módulos → promesas/async-await → optional chaining → array methods → JSON. Cierre con teoría profunda: event loop, closures, gotchas. Cada bloque enseña un trozo de sintaxis que el siguiente reusará.

1. Variables: const y let (olvida var)

Tres formas históricas: var (función-scoped, obsoleto), let (block-scoped, reasignable) y const (block-scoped, no reasignable). En código moderno: siempre const; sólo let cuando vas a reasignar.

const

const API_BASE = '/api';
const TAMANYO_PAGINA = 9;

let

let url = `${API_BASE}${path}`;
if (qs) url += `?${qs}`;
const no es "inmutable", es "no reasignable". Puedes mutar el objeto/array al que apunta: const a = []; a.push(1); es válido. Lo prohibido es const a = []; a = [1];.

2. Arrow functions =>

Sintaxis corta para funciones. Aparece en todo el proyecto y es la base de los hooks (que reciben callbacks).

function suma(a, b) {
  return a + b;
}
// Igual de potente, más corta
const suma = (a, b) => a + b;

// Con cuerpo en bloque (vuelve el return explícito)
const suma2 = (a, b) => {
  const r = a + b;
  return r;
};
// src/api/usuarios.js
export const crearUsuario = (datos) =>
  request('/usuarios', { method: 'POST', body: datos });

export const obtenerUsuario = (id) => request(`/usuarios/${id}`);

Diferencia clave: las arrow no tienen su propio this; heredan el del entorno. Por eso son más seguras como callbacks. En React funcional casi no se usa this, así que rara vez te morderá.

Si la función es de una sola expresión y devuelve algo: usa arrow sin llaves y sin return. Si necesita varias instrucciones: arrow con llaves y return explícito.

3. Template literals (backticks)

Strings con interpolación usando ` y ${ ... }. Adiós concatenaciones feas.

src/api/client.js 
let url = `${API_BASE}${path}`;
if (qs) url += `?${qs}`;
const error = new Error(`Error ${respuesta.status} en ${method} ${path}`);

Equivalente sin template:

var url = API_BASE + path;
if (qs) url += '?' + qs;
var error = new Error('Error ' + respuesta.status + ' en ' + method + ' ' + path);

4. Destructuring

Sacar propiedades de un objeto o un array a variables, en una línea. Aparece en TODOS los componentes y en casi toda la API.

const persona = { nombre: 'Ana', edad: 30 };
const { nombre, edad } = persona;

// Renombrando
const { nombre: n } = persona;

// Con valor por defecto
const { ciudad = 'Murcia' } = persona;
const colores = ['rojo', 'verde', 'azul'];
const [primero, , tercero] = colores;
// primero = 'rojo', tercero = 'azul'

// Idéntico al patrón [valor, setter] = useState(...)
const [contador, setContador] = useState(0);
// Recibe un objeto con opciones (named arguments)
function pedir({ method = 'GET', body } = {}) {
  console.log(method, body);
}
pedir({ method: 'POST', body: { x: 1 } });
// src/api/client.js
export const request = async (
  path,
  { method = 'GET', body, headers = {}, query, returnLocation = false } = {}
) => { ... };

// src/components/RutaProtegida.jsx
function RutaProtegida({ children, soloAdmin = false }) {
  const { usuario, esAdmin } = useAuth();
}
{ } a la izquierda de = ⇒ destructuring de objeto. [ ] a la izquierda ⇒ destructuring de array. Sirve también en argumentos de función: si el llamador pasa {a:1, b:2}, tú lo recibes ya descompuesto.

5. Spread ... y rest ...

Los tres puntos hacen dos cosas opuestas según dónde estén: spread expande, rest agrupa.

Spread: expandir

patrones básicos 
// Combinar arrays
const a = [1, 2];
const b = [...a, 3, 4]; // [1, 2, 3, 4]

// Copiar y modificar objetos (clave en React: nunca mutar)
const persona = { nombre: 'Ana', edad: 30 };
const mayor = { ...persona, edad: 31 }; // { nombre: 'Ana', edad: 31 }

// Mezclar defaults con overrides
const opciones = { ...defaults, ...sobreescrituras };
src/api/client.js — caso real 
const opciones = {
  method,
  headers: {
    Accept: 'application/json',
    ...(body ? { 'Content-Type': 'application/json' } : {}),  // spread condicional
    ...(token ? { Authorization: `Bearer ${token}` } : {}),
    ...headers,
  },
};

Rest: agrupar

function suma(...nums) {
  return nums.reduce((a, b) => a + b, 0);
}
suma(1, 2, 3, 4); // 10

// En destructuring: separa la primera prop y agrupa el resto
const { id, ...resto } = { id: 1, nombre: 'Ana', edad: 30 };
// resto = { nombre: 'Ana', edad: 30 }
CRUD inmutable sobre arrays/objetos

En React, modificar estado SIEMPRE crea valores nuevos. Spread es la herramienta:

// Create: añade un producto a la lista del estado
setProductos([...productos, nuevo]);

// Read: simplemente leer (no muta, no necesita spread)
const item = productos.find(p => p.id === id);

// Update: cambia uno por su id
setProductos(productos.map(p => p.id === id ? { ...p, ...cambios } : p));

// Delete: filtra fuera el que no quieres
setProductos(productos.filter(p => p.id !== id));

Recuerda este patrón: lo verás veinte veces en el proyecto.

6. Módulos: import / export

Cada .js/.jsx es un módulo independiente. Para usar algo de otro fichero hay que exportarlo allí e importarlo aquí. Sin esto, el código sería un solo fichero gigante.

Export named

Varios por fichero. Importas por nombre exacto.

// math.js
export const PI = 3.14;
export function suma(a, b) { return a + b; }

// otra.js
import { PI, suma } from './math';

Export default

Uno solo por fichero. Lo importas con el nombre que quieras.

// Saludo.jsx
export default function Saludo() { return <p>Hola</p>; }

// App.jsx
import Saludo from './Saludo'; // sin llaves
src/pages/Productos.jsx — variantes que verás 
import { useEffect, useState } from 'react';            // named de paquete npm
import { useSearchParams, Link } from 'react-router-dom';
import { Container, Row, Col } from 'react-bootstrap';
import * as productosApi from '../api/productos';      // ⭐ namespace import
import ProductoCard from '../components/ProductoCard'; // default
import './Productos.css';                              // side-effect (no variable)
import * as foo agrupa todos los exports named bajo un alias. Útil cuando un módulo tiene 5+ funciones (api/productos.js). Evita largas listas de imports.

7. Promesas y async / await

Las operaciones que tardan (HTTP, I/O) son asíncronas: no devuelven el resultado al instante, sino una Promise que se "cumplirá" o "fallará" luego. Una Promise es una máquina de estados:

Estilo encadenado (.then/.catch/.finally)

src/pages/Productos.jsx — fragmento 
productosApi
  .listarProductos({ ...filtros, page, size: TAMANYO_PAGINA })
  .then((res) => {
    setProductos(desempaquetar(res, 'productoList'));
    setTotalPaginas(totalPaginasRes(res));
  })
  .catch((err) => setError(err.message))
  .finally(() => setCargando(false));

Estilo async/await (más legible)

src/pages/DetalleProducto.jsx — fragmento 
const comprar = async () => {
  if (!usuario) {
    navegar('/login');
    return;
  }
  try {
    await compraventasApi.crearCompraventa({
      idProducto: producto.id,
      idComprador: usuario.id,
    });
    setMensaje('¡Compra solicitada con éxito!');
    const refrescado = await productosApi.obtenerProducto(id);
    setProducto(refrescado);
  } catch (err) {
    setError(err.message);
  }
};

Reglas de oro

  1. Una función async SIEMPRE devuelve una Promise.
  2. await sólo dentro de async (o top-level de módulo).
  3. await pX "pausa" la función hasta que la Promise resuelva.
  4. Errores: try/catch o .catch().
Si el callback de useEffect es async, falla. useEffect espera o undefined o una función de limpieza; una Promise no le sirve. Solución: declara una función async DENTRO y llámala.

8. Optional chaining ?. y nullish coalescing ??

?. — "si no es null/undefined, accede"

src/components/ProductoCard.jsx 
{producto.lugarDeRecogida?.recogida && ` · ${producto.lugarDeRecogida.recogida}`}
// Si lugarDeRecogida es undefined o null, NO da error. Devuelve undefined.

?? — "si es null/undefined, usa el valor por defecto"

src/api/util.js 
export const totalPaginas = (res) => res?.page?.totalPages ?? 1;
?? sólo trata null y undefined como vacíos. || también 0, '', false. Si tienes precios donde 0 es legítimo, usa ?? siempre. 0 || 5 = 5 ; 0 ?? 5 = 0.

12. Teoría profunda que el entrevistador explota

12.1 El event loop: por qué JS es asíncrono siendo single-threaded

JavaScript tiene UNA sola pila de llamadas. En un instante sólo ejecuta una cosa. ¿Cómo no bloquea la pantalla mientras espera un fetch? El event loop:

Cuando escribes await fetch('/api/productos'):

  1. La función se "pausa"; la petición la atiende el motor de red (fuera del event loop).
  2. El event loop sigue procesando clicks, timers, otros componentes.
  3. Cuando llega la respuesta, el motor de red encola el callback en la microtask queue.
  4. El event loop reanuda tu función desde donde la dejó.
Las microtasks (Promises) tienen prioridad sobre las macrotasks (setTimeout). Por eso un Promise.resolve().then(...) se ejecuta ANTES que un setTimeout(..., 0).
Demuestra el orden

12.2 Closures: la base invisible de los hooks

Un closure es una función que "recuerda" el scope en que fue creada, incluso cuando ese scope ya no está activo en la call stack.

Closure clásico 
function crearContador() {
  let cuenta = 0;
  return function sumar() {
    cuenta++;
    return cuenta;
  };
}
const contador = crearContador();
console.log(contador()); // 1
console.log(contador()); // 2
console.log(contador()); // 3   ← cuenta sigue viva gracias al closure

En DaWeb, cuando loginConGithub crea un setInterval, el callback captura intervaloRef, aceptarToken y navegar. Aunque loginConGithub ya terminó, el closure mantiene esas referencias vivas:

src/pages/Login.jsx — closure en setInterval 
const loginConGithub = () => {
  const popup = window.open('/oauth2/authorization/github', ...);
  intervaloRef.current = setInterval(() => {
    // Esta arrow recuerda popup, intervaloRef, aceptarToken, navegar
    if (popup.closed) { clearInterval(intervaloRef.current); return; }
    aceptarToken(datos.token);
    navegar('/');
  }, 400);
  // loginConGithub termina aquí; el intervalo sigue vivo
};
Cada render de un componente React crea closures NUEVOS con las variables de ese render. Si un efecto "se quedó con datos viejos", probablemente es un stale closure: añade la variable a las dependencias.

12.3 Short-circuit en JSX: el gotcha del 0

// ❌ PELIGRO: si productos.length es 0, React renderiza el 0 en pantalla
{productos.length && <Lista productos={productos} />}

// ✅ Comparación explícita
{productos.length > 0 && <Lista productos={productos} />}

// ✅ Ternario

12.4 Object.fromEntries: convertir array de pares en objeto

src/api/productos.js — caso real 
const ETIQUETAS_ESTADO = Object.fromEntries(
  ESTADOS_PRODUCTO.map((e) => [e.valor, e.etiqueta])
);
export const etiquetaEstado = (valor) => ETIQUETAS_ESTADO[valor] ?? valor;

.map(e => [e.valor, e.etiqueta]) transforma a array de pares ['NUEVO', 'Nuevo']. fromEntries los convierte en objeto { NUEVO: 'Nuevo', ... }. Ahora etiquetaEstado('NUEVO') es O(1) (lookup directo), en vez de O(n) (recorrer con .find()). Cada renderizado de la lista de productos lo agradece.

SOLID · OCP (Open/Closed)

Para añadir un nuevo estado de producto sólo añades una entrada al array ESTADOS_PRODUCTO. etiquetaEstado no se toca; sigue siendo válida. La función está cerrada a modificación pero abierta a extensión a través de su tabla de datos.

// Sin OCP: cada estado nuevo te obliga a tocar la función
export const etiquetaEstado = (v) => {
  if (v === 'NUEVO') return 'Nuevo';
  if (v === 'COMO_NUEVO') return 'Como nuevo';
  // ... 5 if más cada vez que añadas un estado
};

// Con OCP: extiendes el array, la función se mantiene
export const ESTADOS_PRODUCTO = [
  { valor: 'NUEVO', etiqueta: 'Nuevo' },
  { valor: 'COMO_NUEVO', etiqueta: 'Como nuevo' },
  // añadir aquí, listo
];

13. Preguntas trampa frecuentes

"¿Qué imprime console.log(0 || 'hola')? ¿Y 0 ?? 'hola'?"

||: devuelve el primer truthy o el último falsy. 0 es falsy → 'hola'. ??: sólo trata null/undefined como vacío. 0 no lo es → 0. Diferencia crítica con precios (un precio de 0 es válido).

"¿Puede una async function lanzar un error síncrono?"

Sí. Si el error ocurre antes del primer await, el runtime lo envuelve en una Promise rechazada. El llamador captura con .catch o try/catch igualmente.

"¿Qué pasa si haces await de un valor que NO es Promise?"

JS lo envuelve en Promise.resolve(valor). await 42 devuelve 42. Útil cuando una función puede devolver valor síncrono o asíncrono indistintamente.

"¿Por qué const o = {}; o.x = 1 no da error?"

const impide reasignar la referencia, no mutar el contenido. Para inmutabilidad real: Object.freeze(o). React no usa freeze; la convención es no mutar y crear objetos nuevos con spread.

Ejercicio 3 Pasar .then a async/await ★★★ Avanzado

En src/pages/Productos.jsx, la carga inicial de categorías es:

useEffect(() => {
  categoriasApi.listarCategoriasRaiz()
    .then((lista) => setCategorias(lista ?? []))
    .catch(() => setCategorias([]));
}, []);

Reescríbelo dentro de una función async interna.

Solución 
useEffect(() => {
  const cargar = async () => {
    try {
      const lista = await categoriasApi.listarCategoriasRaiz();
      setCategorias(lista ?? []);
    } catch {
      setCategorias([]);
    }
  };
  cargar();
}, []);

Nota: el callback de useEffect NO puede ser async directamente (debe devolver función de limpieza o undefined). Por eso definimos función interna y la llamamos.
