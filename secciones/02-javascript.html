JavaScript moderno (ES6+)

Las características del lenguaje que verás en cada fichero del proyecto.

React y todo el código de DaWeb usa la sintaxis moderna de JavaScript (la que llaman ES6+, ES2015 en adelante). En esta sección verás todas las construcciones que aparecen en el proyecto, con ejemplos extraídos del código real para que en la entrevista las reconozcas al vuelo.

1. Variables: const y let

Hay tres formas de declarar variables: var (antigua, no usar), let (variable normal) y const (variable que no se reasigna).

const

La referencia no cambia. Por defecto siempre usa const.

const API_BASE = '/api';
const TAMANYO_PAGINA = 9;

let

Cuando sí necesitas reasignar.

let url = `${API_BASE}${path}`;
if (query) url += `?${qs}`;
Cuidado

const impide reasignar la referencia, pero no impide modificar el contenido de un objeto o array. const a = []; a.push(1); es válido.

2. Arrow functions =>

Sintaxis corta para funciones. Aparece en todo el proyecto.

function suma(a, b) {
  return a + b;
}
// Igual de potente, más corta
const suma = (a, b) => a + b;

// Con cuerpo en bloque
const suma2 = (a, b) => {
  const r = a + b;
  return r;
};
// src/api/usuarios.js
export const crearUsuario = (datos) =>
  request('/usuarios', { method: 'POST', body: datos });

export const obtenerUsuario = (id) => request(`/usuarios/${id}`);

Diferencia clave: el this

Las arrow functions no tienen su propio this: heredan el del entorno. Por eso son más seguras en callbacks. En React funcional apenas se usa this, así que esto rara vez te morderá.

5. Spread ... y rest ...

Los tres puntos sirven para dos cosas opuestas dependiendo del contexto.

Spread: expandir

ejemplos 
// Combinar arrays
const a = [1, 2];
const b = [...a, 3, 4]; // [1, 2, 3, 4]

// Copiar y modificar objetos (¡muy importante en React!)
const persona = { nombre: 'Ana', edad: 30 };
const mayor = { ...persona, edad: 31 };
// { nombre: 'Ana', edad: 31 }

// Mezclar opciones por defecto con overrides
const opciones = { ...defaults, ...sobreescrituras };
src/api/client.js — uso real 
const opciones = {
  method,
  headers: {
    Accept: 'application/json',
    ...(body ? { 'Content-Type': 'application/json' } : {}),
    ...(token ? { Authorization: `Bearer ${token}` } : {}),
    ...headers,
  },
};

Rest: agrupar

ejemplos 
function suma(...nums) {
  return nums.reduce((a, b) => a + b, 0);
}
suma(1, 2, 3, 4); // 10

const { id, ...resto } = { id: 1, nombre: 'Ana', edad: 30 };
// resto = { nombre: 'Ana', edad: 30 }

6. Módulos: import / export

Cada fichero .js/.jsx es un módulo independiente. Para usar algo de otro fichero hay que exportarlo allí e importarlo aquí.

Export named

Cualquier número de exports por nombre.

// math.js
export const PI = 3.14;
export function suma(a, b) { return a + b; }

// otra.js
import { PI, suma } from './math';

Export default

Sólo uno por fichero. Lo importas con cualquier nombre.

// Saludo.jsx
export default function Saludo() { return <p>Hola</p>; }

// App.jsx
import Saludo from './Saludo';   // sin llaves

Variantes que verás

ejemplos del proyecto 
// src/pages/Productos.jsx
import { useEffect, useState } from 'react';           // named de paquete npm
import { useSearchParams, Link } from 'react-router-dom';
import { Container, Row, Col } from 'react-bootstrap';
import * as productosApi from '../api/productos';      // ⭐ importa TODO como namespace
import * as categoriasApi from '../api/categorias';
import ProductoCard from '../components/ProductoCard'; // default sin llaves
import './Productos.css';                              // efecto secundario (no variable)
El truco del import * as

Cuando un módulo exporta muchas funciones (como api/productos.js), se importa todo bajo un alias para no llenar el fichero de imports. Luego se usa productosApi.listarProductos(...), productosApi.crearProducto(...), etc.

7. Promesas y async / await

Las operaciones que tardan (peticiones HTTP, leer archivos…) son asíncronas: no devuelven el resultado al instante, sino una Promesa que se "cumplirá" más tarde.

Con .then() y .catch()

src/pages/Productos.jsx — fragmento 
productosApi
  .listarProductos({ ...filtros, page, size: TAMANYO_PAGINA })
  .then((res) => {
    setProductos(desempaquetar(res, 'productoList'));
    setTotalPaginas(totalPaginasRes(res));
  })
  .catch((err) => setError(err.message))
  .finally(() => setCargando(false));

Con async/await (más legible)

src/pages/DetalleProducto.jsx — fragmento 
const comprar = async () => {
  if (!usuario) {
    navegar('/login');
    return;
  }
  try {
    await compraventasApi.crearCompraventa({
      idProducto: producto.id,
      idComprador: usuario.id,
    });
    setMensaje('¡Compra solicitada con éxito!');
    const refrescado = await productosApi.obtenerProducto(id);
    setProducto(refrescado);
  } catch (err) {
    setError(err.message);
  }
};

Reglas

  1. Una función marcada con async SIEMPRE devuelve una Promesa.
  2. await sólo se puede usar dentro de una función async.
  3. await pX "pausa" la función hasta que la promesa se resuelva.
  4. Errores: envuelve en try / catch o encadena .catch(...).

8. Optional chaining ?. y nullish coalescing ??

?. — "si no es null/undefined, accede"

src/components/ProductoCard.jsx 
{producto.lugarDeRecogida?.recogida && ` · ${producto.lugarDeRecogida.recogida}`}
// si lugarDeRecogida es undefined o null, NO da error. Devuelve undefined.

?? — "si es null/undefined, usa el valor por defecto"

src/api/util.js 
export const totalPaginas = (res) => res?.page?.totalPages ?? 1;

Distinto de ||: 0 || 5 da 5, pero 0 ?? 5 da 0. ?? sólo "cae" con null/undefined.

Ejercicio 3 Pasar .then a async/await ★★★ Avanzado

En src/pages/Productos.jsx, la carga inicial de categorías es:

useEffect(() => {
  categoriasApi.listarCategoriasRaiz()
    .then((lista) => setCategorias(lista ?? []))
    .catch(() => setCategorias([]));
}, []);

Reescríbelo dentro de una función async interna. Comprueba que sigue funcionando.

Solución 
useEffect(() => {
  const cargar = async () => {
    try {
      const lista = await categoriasApi.listarCategoriasRaiz();
      setCategorias(lista ?? []);
    } catch {
      setCategorias([]);
    }
  };
  cargar();
}, []);

Nota: el callback de useEffect NO puede ser async directamente (debe devolver una función de limpieza o undefined, no una Promesa). Por eso definimos una función interna y la llamamos.
