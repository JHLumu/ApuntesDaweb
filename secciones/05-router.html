React Router: navegación sin recargar

Cómo DaWeb cambia entre URLs sin que el navegador haga un solo viaje extra al servidor.

DaWeb es una SPA: el navegador carga UNA SOLA vez el HTML y luego JavaScript decide qué pintar según la URL. La librería que orquesta eso es react-router-dom. En esta sección vamos del modelo mental (cómo cambia una URL sin recargar) al mapa real de rutas del proyecto, las anidadas con Outlet, y los gotchas de producción.

1. SPA vs página tradicional

Web tradicional

Cada link recarga: petición HTTP al servidor por nuevo HTML. La pantalla parpadea, el estado JS se pierde.

SPA con React Router

El navegador NO recarga. React Router intercepta los clicks, cambia la URL con la History API y pinta el componente que toca. Instantáneo y con estado preservado.

<Link to> evita la recarga; <a href> la fuerza. En una SPA, casi siempre Link. Reserva <a> para enlaces externos al dominio.

4. El esqueleto: App.jsx

src/App.jsx 
function App() {
  return (
    <AuthProvider>
      <BrowserRouter>
        <Layout />
      </BrowserRouter>
    </AuthProvider>
  );
}

Tres envoltorios:

  1. AuthProvider da el contexto de auth a TODA la app (sección 07).
  2. BrowserRouter escucha cambios de URL.
  3. Layout añade header/footer y contiene los Routes.
AuthProvider envuelve BrowserRouter, no al revés. Así RutaProtegida puede usar simultáneamente useAuth() y useNavigate().

El componente Layout

src/App.jsx — Layout 
function Layout() {
  const { pathname } = useLocation();
  const sinLayout = pathname === '/login' || pathname === '/registro';

  return (
    <>
      {!sinLayout && <Header />}
      <main>
        <Routes> ... </Routes>
      </main>
      {!sinLayout && <Footer />}
    </>
  );
}

5. El mapa completo de rutas

src/App.jsx — Routes 
<Routes>
  {/* Públicas */}
  <Route path="/" element={<Home />} />
  <Route path="/login" element={<Login />} />
  <Route path="/registro" element={<Registro />} />
  <Route path="/productos" element={<Productos />} />
  <Route path="/productos/:id" element={<DetalleProducto />} />

  {/* Protegidas */}
  <Route path="/vender" element={
    <RutaProtegida><NuevoProducto /></RutaProtegida>
  } />
  <Route path="/productos/:id/editar" element={
    <RutaProtegida><EditarProducto /></RutaProtegida>
  } />

  {/* Anidadas: /perfil/* */}
  <Route path="/perfil" element={
    <RutaProtegida><Perfil /></RutaProtegida>
  }>
    <Route index element={<Navigate to="productos" replace />} />
    <Route path="productos" element={<PerfilProductos />} />
    <Route path="ventas"    element={<PerfilVentas />} />
    <Route path="compras"   element={<PerfilCompras />} />
  </Route>

  <Route path="/usuarios/:id" element={
    <RutaProtegida><PerfilUsuario /></RutaProtegida>
  } />

  {/* Sólo admin */}
  <Route path="/admin/usuarios" element={
    <RutaProtegida soloAdmin><AdminUsuarios /></RutaProtegida>
  } />
  <Route path="/admin/transacciones" element={
    <RutaProtegida soloAdmin><AdminTransacciones /></RutaProtegida>
  } />

  {/* Catch-all 404 */}
  <Route path="*" element={<Error404 />} />
</Routes>
SOLID · OCP (Open/Closed)

Para añadir una página nueva (por ejemplo /favoritos) sólo añades UNA línea <Route>. El router en sí no se modifica; se extiende. Los componentes existentes no se tocan. Cerrado a modificación, abierto a extensión.

6. Parámetros de URL: useParams

Cuando un path lleva :id, React Router lo expone con useParams:

src/pages/DetalleProducto.jsx 
import { useParams } from 'react-router-dom';

function DetalleProducto() {
  const { id } = useParams();        // /productos/42 → id = "42"

  useEffect(() => {
    productosApi.obtenerProducto(id).then(setProducto);
  }, [id]);
}
Los params siempre llegan como string. Si tu backend pide número, convierte: Number(id). JS no convierte solo.

7. Query strings: useSearchParams

src/pages/Productos.jsx 
import { useSearchParams } from 'react-router-dom';

function Productos() {
  const [searchParams] = useSearchParams();
  const descripcionUrl = searchParams.get('descripcion') ?? '';
}

Esto es lo que permite que el buscador del header navegue a /productos?descripcion=bici y la página coja ese texto como filtro inicial.

8. Navegación: <Link> vs useNavigate()

Link / NavLink — declarativo

Para enlaces que el usuario hace click.

<Link to={`/productos/${p.id}`}>Ver detalle</Link>

<NavLink to="/productos"
  className={({ isActive }) => isActive ? 'activo' : ''}>
  Productos
</NavLink>

useNavigate — imperativo

Para navegar como consecuencia de algo (submit OK, login).

const navegar = useNavigate();

if (res.ok) navegar('/');
navegar(-1);                          // atrás
navegar('/productos', { replace: true });

9. Rutas anidadas y <Outlet>

<Outlet /> es el "hueco" donde React Router inserta la ruta hija. Si Perfil es el padre y PerfilVentas el hijo, sin Outlet el hijo no aparecería aunque matchee.

src/pages/Perfil.jsx — estructura 
function Perfil() {
  return (
    <Container>
      <Row>
        <Col md={4}>
          <FormularioPerfil />
        </Col>
        <Col md={8}>
          <div className="perfil-tabs">
            <NavLink to="/perfil/productos">Mis productos</NavLink>
            <NavLink to="/perfil/ventas">Mis ventas</NavLink>
            <NavLink to="/perfil/compras">Mis compras</NavLink>
          </div>
          <Outlet /> {/* aquí va PerfilProductos / Ventas / Compras */}
        </Col>
      </Row>
    </Container>
  );
}

La ruta index

<Route index element={<Navigate to="productos" replace />} />

index no es un path: es la coincidencia exacta del padre. Si vas a /perfil sin subruta, redirige a /perfil/productos. Sin esta línea, el <Outlet/> estaría vacío.

10. La ruta comodín *

<Route path="*" element={<Error404 />} />

* matchea cualquier URL que no haya matcheado antes. Tu 404.

11. Flujo completo de un click en "Ver detalle"

1 Usuario en /productos click en <Link to="/productos/42">.
2 Link cancela el comportamiento por defecto y llama a la History API.
3 URL cambia a /productos/42 sin recargar.
4 BrowserRouter notifica el cambio. Routes matchea /productos/:id.
5 Se monta <DetalleProducto />. useParams() devuelve { id: "42" }. useEffect pide el producto.

12. La History API por debajo

El secreto de una SPA es la History API de HTML5:

// Cambia URL sin recargar ni ir al servidor
window.history.pushState({}, '', '/productos');

// Reemplaza la entrada actual (no añade)
window.history.replaceState({}, '', '/login');

window.history.back();

React Router envuelve esto en componentes y hooks. BrowserRouter escucha el evento popstate (botón atrás/adelante) y decide qué pintar.

La trampa del refresh en producción

Bug clásico

En prod, si recargas /productos/42, el servidor busca el fichero físico y da 404. Solución: configurar nginx para servir index.html en cualquier ruta:

location / {
  try_files $uri $uri/ /index.html;
}

13. Preguntas trampa frecuentes

"navegar('/') vs navegar('/', { replace: true })"

Sin replace: añade entrada al historial; "atrás" vuelve. Con replace: sustituye la entrada actual. Si un usuario sin login va a /perfil y RutaProtegida lo manda a /login con replace, al pulsar "atrás" vuelve a la página de antes, no a /perfil (que lo redirigiría otra vez → bucle).

<Navigate replace> SIEMPRE en flujos de auth. Evita el bucle del botón "atrás".
"¿Por qué AuthProvider envuelve BrowserRouter y no al revés?"

Para que el contexto de auth esté disponible en cualquier ruta, incluso en RutaProtegida, que necesita useAuth() y useNavigate() a la vez.
