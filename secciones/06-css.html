CSS y diseño responsive

Cómo se construye el aspecto visual de DaWeb y cómo se adapta a móviles.

El diseño de DaWeb se sostiene sobre cuatro pilares: Bootstrap 5 (sistema de columnas y utilidades), variables CSS propias (paleta lila/lavanda en theme.css), CSS Grid (cuadrículas auto-ajustables) y Media Queries (para móvil). En esta sección los recorremos uno a uno con código real del proyecto.

Por qué inline en React es un objeto

Las dos llaves son: las del JSX para meter una expresión, y las del objeto JavaScript. Las propiedades van en camelCase (backgroundColor, no background-color).

¿Por qué variables y no colores escritos a mano?

Para cambiar TODO el tema visual con un solo edit. Si reemplazas --primary por un verde, todas las cards, botones y bordes se vuelven verdes sin tocar nada más.

Clases utilitarias del proyecto

En theme.css hay tres clases muy usadas:

  • .card-hover: cuando pasas el ratón, la tarjeta sube y se le aumenta la sombra.
  • .seccion-titulo: título con una barra lila a la izquierda. La verás en casi todas las páginas.

3. Bootstrap 5: el sistema de columnas

Bootstrap divide la pantalla en 12 columnas. Para layout, usas Container > Row > Col.

src/pages/DetalleProducto.jsx 
<Container className="py-4">
  <Row className="g-4">
    <Col lg={7}>
      <Card>...</Card>
    </Col>
    <Col lg={5}>
      <Card>...</Card>
    </Col>
  </Row>
</Container>

Lectura: en pantallas grandes (lg y más), la primera columna ocupa 7/12 (60%) y la segunda 5/12 (40%). En móvil (por debajo de lg), Bootstrap apila ambas a 12/12.

4. CSS Grid: cuadrículas inteligentes

Bootstrap es bueno para layouts globales pero para listas de productos hay algo mejor: CSS Grid. El proyecto la usa en la página de productos:

src/pages/Productos.css 
.productos-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(16rem, 1fr));
  gap: 1.5rem;
}

@media (max-width: 576px) {
  .productos-grid {
    grid-template-columns: 1fr;
  }
}

Lectura de repeat(auto-fill, minmax(16rem, 1fr)):

  • auto-fill: mete cuantas columnas quepan.
  • minmax(16rem, 1fr): cada columna mide entre 16rem (256px) y 1 fracción del espacio.

Resultado: en una pantalla grande caben 4-5 cards, en una mediana 3, en una pequeña 2, y en móvil una. Sin escribir media queries para cada tamaño. Sólo añadimos la regla extra para móvil con una columna explícita.

5. Media Queries

Una media query es "si el navegador cumple esta condición, aplica estas reglas".

src/Home.css — fragmento 
@media (max-width: 991.98px) {
  .home-hero__titulo {
    font-size: 2.4rem;     /* en pantallas medianas el título es más pequeño */
  }
}

@media (max-width: 767.98px) {
  .marquee {
    grid-template-columns: 1fr;
    height: auto;
  }
  .marquee__track--down { display: none; }
}

@media (prefers-reduced-motion: reduce) {
  .marquee__track--up,
  .marquee__track--down {
    animation: none;     /* accesibilidad: desactivar animaciones */
  }
}

6. Animaciones con @keyframes

La home tiene un carrusel vertical infinito ("marquee"). Está hecho con animación CSS:

src/Home.css 
.marquee__track--up {
  animation: marquee-up 24s linear infinite;
}

@keyframes marquee-up {
  from { transform: translateY(0); }
  to   { transform: translateY(-50%); }
}

Lectura: durante 24s, traslada el contenido un -50% en Y. Como el contenido se duplica en JSX ([...BLOQUES, ...BLOQUES]), al llegar al -50% el siguiente fotograma se ve exactamente igual que el primero, creando ilusión de bucle.

7. Estructura de estilos del proyecto

1 main.jsx importa Bootstrap, index.css y theme.css globales.
2 index.css define tipografía base y reset.
3 theme.css define variables y clases utilitarias propias.
4 Cada página o componente que necesita estilos específicos importa su propio .css.

8. Cómo Bootstrap convive con tus estilos personalizados

El orden importa. En main.jsx:

main.jsx 
import 'bootstrap/dist/css/bootstrap.min.css'  // 1º Bootstrap
import './index.css'                            // 2º base del proyecto
import './theme.css'                            // 3º tema del proyecto

Los estilos posteriores sobrescriben los anteriores cuando tienen la misma especificidad. Por eso .btn-lav definido en theme.css tiene precedencia sobre los botones por defecto de Bootstrap.

9. Teoría profunda: lo que el entrevistador sabe

La especificidad CSS: qué regla "gana"

Cuando dos reglas CSS apuntan al mismo elemento, gana la más específica. La especificidad se calcula como (A, B, C):

SelectorA (IDs)B (clases, attrs)C (etiquetas)Valor
div001(0,0,1)
.btn-lav010(0,1,0)
button.btn-lav011(0,1,1)
#header100(1,0,0)
style="" inline———siempre gana

Bootstrap usa clases (.btn, .btn-primary), por eso para sobreescribir puedes añadir más especificidad o cargarlo después con la misma:

/* Bootstrap define: .btn-primary { background: #0d6efd } — especificidad (0,1,0) */

/* Override con misma especificidad pero DESPUÉS en el orden de carga */
.btn-lav { background: var(--ink); }  /* también (0,1,0) — funciona si carga después */

/* Override más específico: siempre gana independientemente del orden */
button.btn-lav { background: var(--ink); }  /* (0,1,1) > (0,1,0) */

En DaWeb, theme.css se carga DESPUÉS de Bootstrap (main.jsx: Bootstrap primero, luego index.css, luego theme.css). Esto garantiza que los estilos personalizados sobreescriben los de Bootstrap con la misma especificidad.

El modelo de caja (box model)

Cada elemento HTML es una caja con cuatro capas:

visualización del box model 
┌─────────────── margin ───────────────┐
│  ┌──────────── border ─────────────┐ │
│  │  ┌──────── padding ───────────┐ │ │
│  │  │      CONTENT (width)       │ │ │
│  │  └────────────────────────────┘ │ │
│  └──────────────────────────────────┘ │
└────────────────────────────────────────┘

Hay dos modos de calcular el width:

  • box-sizing: content-box (por defecto CSS): width = solo el contenido. Si añades padding, el elemento se hace más ancho que el width declarado.
  • box-sizing: border-box (Bootstrap lo impone globalmente): width incluye padding y border. Mucho más intuitivo: width: 300px siempre mide 300px.
/* Bootstrap hace esto globalmente */
*, *::before, *::after {
  box-sizing: border-box;
}

/* Efecto: col con width:300px y padding:16px
   tiene content de 300-32=268px (no 332px desbordando el layout) */

Flexbox: el corazón del layout de Bootstrap

El sistema de grid de Bootstrap (Row/Col) se basa en Flexbox. Entender Flexbox es entender por qué d-flex gap-2 align-items-center funciona:

/* Contenedor flex (el padre) */
.d-flex { display: flex; }
/* Por defecto: flex-direction: row, flex-wrap: nowrap */

/* Alineación en el eje principal (horizontal si row) */
.justify-content-between { justify-content: space-between; }
.justify-content-center  { justify-content: center; }

/* Alineación en el eje cruzado (vertical si row) */
.align-items-center  { align-items: center; }
.align-items-stretch { align-items: stretch; } /* por defecto */

/* Separación entre items */
.gap-2 { gap: 0.5rem; }

En DaWeb, en el header de sesión iniciada:

<div className="d-flex gap-2 align-items-center">
  <button className="header-btn" onClick={salir}>Salir</button>
  <span>{usuario.nombre}</span>
</div>

El div es un contenedor flex: sus hijos (botón y span) se colocan en fila, alineados verticalmente al centro, con 0.5rem de separación.

Las variables CSS y la cascada: por qué cambia toda la app

Las variables CSS se definen en :root. El selector :root es el elemento más alto del árbol (equivale a <html>). Las variables se heredan en cascada hacia todos los descendientes — es decir, hacia toda la página:

/* En :root = accesible desde cualquier elemento de la página */
:root { --primary: #c8b6ff; }

.card { border-color: var(--primary); }
.seccion-titulo { border-left-color: var(--primary); }
/* Cambiar --primary en :root cambia AMBAS reglas automáticamente */

/* Sobreescribir para un subtree local */
.seccion-oscura {
  --primary: #8a2be2; /* solo aplica dentro de .seccion-oscura */
}

Animaciones con @keyframes: el truco del bucle infinito

La animación marquee de Home.jsx usa este truco para el bucle perfecto:

  1. El contenido se duplica en JSX: [...BLOQUES, ...BLOQUES] → 8 items en lugar de 4.
  2. La animación CSS mueve el track un -50% en Y (equivale a desplazar los 4 primeros fuera de vista).
  3. Al llegar al -50%, los 4 segundos son idénticos a los 4 primeros → el siguiente fotograma es visualmente igual al inicio → bucle invisible sin parpadeo.
@keyframes marquee-up {
  from { transform: translateY(0); }    /* posición inicial */
  to   { transform: translateY(-50%); } /* -50% = el primer grupo sale, el segundo ocupa su lugar */
}

/* Contenido: [A, B, C, D, A, B, C, D]
   At -50%:  los A,B,C,D originales están fuera de la ventana,
             los A,B,C,D de la copia están en la posición inicial
   → idéntico al inicio → bucle invisible */
Pregunta trampa del entrevistador

"¿Por qué Bootstrap usa border-box como box-sizing?" — Porque el comportamiento por defecto (content-box) es contraintuitivo: si pones width: 100% y añades padding, el elemento desborda su contenedor. Con border-box, el ancho incluye el padding, así width: 100% con padding siempre encaja sin desbordar.

Pregunta trampa del entrevistador

"¿Qué es la cascada CSS y por qué importa el orden de importación?" — Cascada = cuando dos reglas tienen la misma especificidad, gana la que aparece después en el CSS. En main.jsx, Bootstrap se importa primero y theme.css después. Así, .card en theme.css sobreescribe .card de Bootstrap sin necesitar mayor especificidad. Si se invirtiera el orden, Bootstrap ganaría.

Pregunta trampa del entrevistador

"¿Qué diferencia hay entre display: none y visibility: hidden?" — display: none elimina el elemento del flujo del documento: no ocupa espacio, no recibe clicks. visibility: hidden lo hace invisible pero sigue ocupando espacio. En Bootstrap, .d-none usa display: none; al añadir .d-md-block cambia a display: block a partir de la breakpoint md.

  1. Abre src/theme.css.
  2. Cambia --primary de #c8b6ff (lavanda) a #1971c2 (azul) o a un color que te guste.
  3. Mira la web: la barra de los títulos, los focus de inputs y los botones lavanda cambian.
Si quieres ir más lejos

Cambia también --ink (texto principal) y --mauve-2 (usado en el hero) para reescribir toda la paleta.

Ejercicio 2 Modificar la grid de productos ★★ Intermedio

En src/pages/Productos.css cambia minmax(16rem, 1fr) por minmax(12rem, 1fr). Recarga /productos y observa: caben más tarjetas por fila porque ahora el mínimo es menor.

Variante

Prueba con minmax(22rem, 1fr): las tarjetas son más anchas y caben menos. Esta sola línea sustituye varias media queries.

}
Ejercicio 4 Crear una nueva clase utility ★★★ Avanzado

Ejercicio 5 Modo oscuro casero ★★★ Avanzado

Añade al final de theme.css:

@media (prefers-color-scheme: dark) {
  :root {
    --surface: #1a1727;
    --surface-soft: #2a2440;
    --ink: #f6f4fb;
    --ink-soft: #c8b6ff;
  }
}

Si tu sistema operativo está en modo oscuro, la web debería invertir colores. Si no, cambia temporalmente el modo del sistema o sustituye @media por un selector como :root[data-tema="oscuro"] y alterna el atributo con JavaScript.
