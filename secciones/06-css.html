CSS y diseño responsive

Cómo se construye el aspecto visual de DaWeb y cómo se adapta a móviles.

El diseño de DaWeb se sostiene sobre cuatro pilares: Bootstrap 5 (sistema de columnas y utilidades), variables CSS propias (paleta lila/lavanda en theme.css), CSS Grid (cuadrículas auto-ajustables) y Media Queries (para móvil). En esta sección los recorremos uno a uno con código real del proyecto.

Por qué inline en React es un objeto

JSX 
<h4 style={{ color: 'var(--ink)' }}>Hola</h4>

Las dos llaves son: las del JSX para meter una expresión, y las del objeto JavaScript. Las propiedades van en camelCase (backgroundColor, no background-color).

2. Variables CSS (custom properties)

Definidas en :root (o en cualquier selector), permiten reusar valores como una paleta. Toda la identidad visual del proyecto vive en theme.css:

src/theme.css 
:root {
  --petal-frost: #ffd6ff;
  --mauve: #e7c6ff;
  --mauve-2: #c8b6ff;
  --periwinkle: #b8c0ff;
  --periwinkle-2: #bbd0ff;

  --surface: #ffffff;
  --surface-soft: #f6f4fb;
  --primary: #c8b6ff;
  --ink: #2a2440;
  --ink-soft: #4a4458;

  --radius-card: 1rem;
  --radius-field: 0.75rem;
  --shadow-soft: 0 6px 20px rgba(122, 102, 168, 0.14);
  --shadow-hover: 0 16px 34px rgba(122, 102, 168, 0.26);
  --transition: 0.25s ease;
}

Y se usan en cualquier sitio con var(--nombre):

src/theme.css — usos 
.card {
  border: none;
  border-radius: var(--radius-card);
  box-shadow: var(--shadow-soft);
}

.btn-lav {
  background: var(--ink);
  color: #fff;
  border-radius: 999px;
  transition: background var(--transition);
}
.btn-lav:hover { background: #3d3559; }

.seccion-titulo {
  border-left: 4px solid var(--primary);
  padding-left: 0.85rem;
  font-weight: 700;
  color: var(--ink);
}
¿Por qué variables y no colores escritos a mano?

Para cambiar TODO el tema visual con un solo edit. Si reemplazas --primary por un verde, todas las cards, botones y bordes se vuelven verdes sin tocar nada más.

Clases utilitarias del proyecto

En theme.css hay tres clases muy usadas:

  • .card-hover: cuando pasas el ratón, la tarjeta sube y se le aumenta la sombra.
  • .seccion-titulo: título con una barra lila a la izquierda. La verás en casi todas las páginas.
  • .btn-lav: el botón principal del proyecto (oscuro, redondeado, ese "look" lavanda).

3. Bootstrap 5: el sistema de columnas

Bootstrap divide la pantalla en 12 columnas. Para layout, usas Container > Row > Col.

src/pages/DetalleProducto.jsx 
<Container className="py-4">
  <Row className="g-4">
    <Col lg={7}>
      <Card>...</Card>
    </Col>
    <Col lg={5}>
      <Card>...</Card>
    </Col>
  </Row>
</Container>

Lectura: en pantallas grandes (lg y más), la primera columna ocupa 7/12 (60%) y la segunda 5/12 (40%). En móvil (por debajo de lg), Bootstrap apila ambas a 12/12.

4. CSS Grid: cuadrículas inteligentes

Bootstrap es bueno para layouts globales pero para listas de productos hay algo mejor: CSS Grid. El proyecto la usa en la página de productos:

src/pages/Productos.css 
.productos-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(16rem, 1fr));
  gap: 1.5rem;
}

@media (max-width: 576px) {
  .productos-grid {
    grid-template-columns: 1fr;
  }
}

Lectura de repeat(auto-fill, minmax(16rem, 1fr)):

  • auto-fill: mete cuantas columnas quepan.
  • minmax(16rem, 1fr): cada columna mide entre 16rem (256px) y 1 fracción del espacio.

Resultado: en una pantalla grande caben 4-5 cards, en una mediana 3, en una pequeña 2, y en móvil una. Sin escribir media queries para cada tamaño. Sólo añadimos la regla extra para móvil con una columna explícita.

5. Media Queries

Una media query es "si el navegador cumple esta condición, aplica estas reglas".

src/Home.css — fragmento 
@media (max-width: 991.98px) {
  .home-hero__titulo {
    font-size: 2.4rem;     /* en pantallas medianas el título es más pequeño */
  }
}

@media (max-width: 767.98px) {
  .marquee {
    grid-template-columns: 1fr;
    height: auto;
  }
  .marquee__track--down { display: none; }
}

@media (prefers-reduced-motion: reduce) {
  .marquee__track--up,
  .marquee__track--down {
    animation: none;     /* accesibilidad: desactivar animaciones */
  }
}

6. Animaciones con @keyframes

La home tiene un carrusel vertical infinito ("marquee"). Está hecho con animación CSS:

src/Home.css 
.marquee__track--up {
  animation: marquee-up 24s linear infinite;
}

@keyframes marquee-up {
  from { transform: translateY(0); }
  to   { transform: translateY(-50%); }
}

Lectura: durante 24s, traslada el contenido un -50% en Y. Como el contenido se duplica en JSX ([...BLOQUES, ...BLOQUES]), al llegar al -50% el siguiente fotograma se ve exactamente igual que el primero, creando ilusión de bucle.

7. Estructura de estilos del proyecto

1 main.jsx importa Bootstrap, index.css y theme.css globales.
2 index.css define tipografía base y reset.
3 theme.css define variables y clases utilitarias propias.
4 Cada página o componente que necesita estilos específicos importa su propio .css.

8. Cómo Bootstrap convive con tus estilos personalizados

El orden importa. En main.jsx:

main.jsx 
import 'bootstrap/dist/css/bootstrap.min.css'  // 1º Bootstrap
import './index.css'                            // 2º base del proyecto
import './theme.css'                            // 3º tema del proyecto

Los estilos posteriores sobrescriben los anteriores cuando tienen la misma especificidad. Por eso .btn-lav definido en theme.css tiene precedencia sobre los botones por defecto de Bootstrap.

10. Ejercicios sobre el proyecto

Ejercicio 1 Cambiar el color primario del tema ★ Muy fácil
  1. Abre src/theme.css.
  2. Cambia --primary de #c8b6ff (lavanda) a #1971c2 (azul) o a un color que te guste.
  3. Mira la web: la barra de los títulos, los focus de inputs y los botones lavanda cambian.
Si quieres ir más lejos

Cambia también --ink (texto principal) y --mauve-2 (usado en el hero) para reescribir toda la paleta.

Ejercicio 2 Modificar la grid de productos ★★ Intermedio

En src/pages/Productos.css cambia minmax(16rem, 1fr) por minmax(12rem, 1fr). Recarga /productos y observa: caben más tarjetas por fila porque ahora el mínimo es menor.

Variante

Prueba con minmax(22rem, 1fr): las tarjetas son más anchas y caben menos. Esta sola línea sustituye varias media queries.

Ejercicio 3 Añadir un breakpoint para tablet ★★ Intermedio

Añade en Productos.css una regla que muestre exactamente 2 columnas entre 576px y 768px:

@media (min-width: 577px) and (max-width: 768px) {
  .productos-grid { grid-template-columns: repeat(2, 1fr); }
}
Ejercicio 4 Crear una nueva clase utility ★★★ Avanzado

Imagina que quieres una clase .btn-peri idéntica a .btn-lav pero en color periwinkle. Defínela en theme.css:

.btn-peri {
  background: var(--periwinkle);
  border: none;
  color: var(--ink);
  border-radius: 999px;
  padding: 0.6rem 1.5rem;
  font-weight: 500;
  transition: background var(--transition);
}
.btn-peri:hover { background: var(--periwinkle-2); }

Y úsala donde quieras: <button className="btn btn-peri">Acción</button>.

Ejercicio 5 Modo oscuro casero ★★★ Avanzado

Añade al final de theme.css:

@media (prefers-color-scheme: dark) {
  :root {
    --surface: #1a1727;
    --surface-soft: #2a2440;
    --ink: #f6f4fb;
    --ink-soft: #c8b6ff;
  }
}

Si tu sistema operativo está en modo oscuro, la web debería invertir colores. Si no, cambia temporalmente el modo del sistema o sustituye @media por un selector como :root[data-tema="oscuro"] y alterna el atributo con JavaScript.
