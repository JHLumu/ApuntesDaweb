Home — src/pages/Home.jsx

Qué hace

Página de bienvenida. Dos secciones:

  1. Hero: título grande, frase descriptiva y tres "estadísticas" decorativas (12.500+ usuarios, 50.000+ productos vendidos, 4,8/5 valoración).
  2. Segunda sección: texto sobre dar segunda vida + un marquee vertical infinito de imágenes de productos.

Implementación destacada

src/pages/Home.jsx — el marquee 
const BLOQUES = [0, 1, 2, 3];

<div className="marquee">

Login — src/pages/Login.jsx

Qué hace

  • Formulario controlado de correo y contraseña.
  • Botón "Iniciar sesión con GitHub" que abre un popup OAuth2.
  • Tras login OK, redirige a la home.
  • Layout especial: ocupa toda la pantalla, sin Header ni Footer (esto se controla en App.jsx).

Estado del componente

Login.jsx — fragmento 
const { login, aceptarToken } = useAuth();
const navegar = useNavigate();
const [correo, setCorreo] = useState('');
const [clave, setClave] = useState('');
const [error, setError] = useState('');
const [enviando, setEnviando] = useState(false);
const intervaloRef = useRef(null);

Submit del formulario

Login.jsx — enviar 
const enviar = async (e) => {
  e.preventDefault();                // ⬅ evita que recargue la página
  setError('');
  setEnviando(true);
  const res = await login(correo, clave);   // función del contexto
  setEnviando(false);
  if (res.ok) navegar('/');
  else setError(res.error || 'No se ha podido iniciar sesión');
};
e.preventDefault()

En HTML estándar, al enviar un form el navegador recarga la página. Como nosotros gestionamos el envío con JS, lo cancelamos.

OAuth con GitHub

Ya explicado en detalle en la sección 07. Resumen: popup → setInterval que vigila cuándo el popup vuelve a tu dominio → leer JSON con el token → aceptarToken.

Ejercicio Validación de campos ★★ Intermedio

Añade validación: la contraseña debe tener al menos 6 caracteres. Antes de llamar a login:

if (clave.length < 6) {
  setError('La contraseña debe tener al menos 6 caracteres');
  setEnviando(false);
  return;
}

Registro — src/pages/Registro.jsx

Qué hace

Formulario con campos: nombre, apellidos, correo, clave, fechaNacimiento, teléfono. Llama a registrar del contexto (que internamente hace POST /usuarios y, si va bien, login automático).

Estado en un solo objeto

Registro.jsx 
const [datos, setDatos] = useState({
  nombre: '', apellidos: '', correo: '',
  clave: '', fechaNacimiento: '', telefono: '',
});

const cambiar = (e) =>
  setDatos({ ...datos, [e.target.name]: e.target.value });

El truco [e.target.name] permite usar UNA función para TODOS los inputs. Cada input tiene su name y la función modifica esa propiedad concreta del objeto.

Validación nativa de HTML

El formulario usa atributos HTML para validar:

  • required: el campo no puede estar vacío.
  • type="email": el navegador valida formato de correo.
  • type="date": muestra un selector de fecha.
  • minLength={4} en la clave.
Ejercicio Añadir confirmación de contraseña ★★ Intermedio

Esqueleto 
if (datos.clave !== datos.claveRepetida) {
  setError('Las contraseñas no coinciden');
  setEnviando(false);
  return;
}

Productos — src/pages/Productos.jsx

Qué hace

Listado paginado de productos con cuatro filtros y un botón "Publicar producto".

Estado

Productos.jsx — estado 
const TAMANYO_PAGINA = 9;
const FILTROS_VACIOS = { descripcion: '', categoria: '', estado: '', precioMaximo: '' };

const [searchParams] = useSearchParams();
const descripcionUrl = searchParams.get('descripcion') ?? '';

const [filtros, setFiltros] = useState({ ...FILTROS_VACIOS, descripcion: descripcionUrl });
const [page, setPage] = useState(0);
const [productos, setProductos] = useState([]);
const [totalPaginas, setTotalPaginas] = useState(1);
const [categorias, setCategorias] = useState([]);
const [cargando, setCargando] = useState(false);
const [error, setError] = useState('');

Tres useEffect

  1. Cargar categorías UNA vez al montar (lista para el desplegable).
  2. Reaccionar a cambios de URL: si llega ?descripcion=xxx desde el header, actualiza el filtro.
  3. Cargar productos cuando cambien filtros o página.
Productos.jsx — efectos 
useEffect(() => {
  categoriasApi.listarCategoriasRaiz()
    .then((lista) => setCategorias(lista ?? []))
    .catch(() => setCategorias([]));
}, []);

useEffect(() => {
  setPage(0);
  setFiltros((prev) => ({ ...prev, descripcion: descripcionUrl }));
}, [descripcionUrl]);

useEffect(() => {
  setCargando(true);
  setError('');
  productosApi
    .listarProductos({ ...filtros, page, size: TAMANYO_PAGINA })
    .then((res) => {
      setProductos(desempaquetar(res, 'productoList'));
      setTotalPaginas(totalPaginasRes(res));
    })
    .catch((err) => setError(err.message))
    .finally(() => setCargando(false));
}, [filtros, page]);

Filtros: cuatro inputs controlados con una sola función

const cambiarFiltro = (e) => {
  setPage(0);                                      // resetea a página 0 al filtrar
  setFiltros({ ...filtros, [e.target.name]: e.target.value });
};

<Form.Control name="descripcion" value={filtros.descripcion} onChange={cambiarFiltro} />
<Form.Select name="categoria"   value={filtros.categoria}   onChange={cambiarFiltro}> ... </Form.Select>
<Form.Select name="estado"      value={filtros.estado}      onChange={cambiarFiltro}> ... </Form.Select>
<Form.Control name="precioMaximo" type="number" value={filtros.precioMaximo} onChange={cambiarFiltro} />
Ejercicio Añadir un quinto filtro ★★ Intermedio

Añade un filtro "precio mínimo":

  1. En FILTROS_VACIOS añade precioMinimo: ''.
  2. Añade otro <Col md={2}> con un Form.Control type="number" name="precioMinimo".
  3. No tocas productos.js: listarProductos ya pasa todos los filtros como query string.
Nota sobre el backend

El backend ArSo tendría que aceptar también ese parámetro. Si no lo hace, el filtro simplemente se ignora.

DetalleProducto — src/pages/DetalleProducto.jsx

Qué hace

Página del producto individual. Muestra:

  • Descripción larga, categoría, fecha de publicación, visualizaciones, envío disponible.
  • Lugar de recogida con enlace a OpenStreetMap si tiene coordenadas.
  • Tarjeta con precio, estado, badge "Vendido" si lo está.
  • Botón "Comprar" (o "Editar" / "Eliminar" si es del usuario).
  • Datos del vendedor con link a su perfil.

Efecto que carga y suma visualización

const visualizacionContadaRef = useRef(null);

useEffect(() => {
  setCargando(true);
  productosApi.obtenerProducto(id)
    .then(setProducto)
    .catch((err) => setError(err.message))
    .finally(() => setCargando(false));

  if (visualizacionContadaRef.current !== id) {
    visualizacionContadaRef.current = id;
    productosApi.sumarVisualizacion(id).catch(() => {});
  }
}, [id]);

El useRef evita sumar dos visualizaciones al mismo producto en el mismo renderizado (porque StrictMode en desarrollo monta dos veces).

Acción "Comprar"

const comprar = async () => {
  if (!usuario) { navegar('/login'); return; }
  try {
    await compraventasApi.crearCompraventa({
      idProducto: producto.id,
      idComprador: usuario.id,
    });
    setMensaje('¡Compra solicitada con éxito!');
    const refrescado = await productosApi.obtenerProducto(id);  // refresca estado
    setProducto(refrescado);
  } catch (err) {
    setError(err.message);
  }
};

Acción "Eliminar"

const eliminar = async () => {
  const confirmado = window.confirm(`¿Seguro que quieres eliminar "${producto.titulo}"?`);
  if (!confirmado) return;
  await productosApi.eliminarProducto(producto.id);
  navegar('/perfil');
};

Renderizado condicional según rol

const esMio = usuario && producto.vendedor?.id === usuario.id;
const yaVendido = producto.vendido;
// ...
{esMio ? (
  !yaVendido && (
    <div>
      <Link to={`/productos/${producto.id}/editar`} className="btn btn-lav">Editar producto</Link>
      <Button variant="outline-danger" onClick={eliminar}>Eliminar producto</Button>
    </div>

) : (
  <button onClick={comprar} disabled={yaVendido}>
    {yaVendido ? 'No disponible' : 'Comprar'}

Enlace al mapa

<a href={`https://www.openstreetmap.org/?mlat=${lat}&mlon=${lon}#map=15/${lat}/${lon}`}

NuevoProducto — src/pages/NuevoProducto.jsx

Qué hace

Formulario para publicar un producto. Campos:

  • Título, descripción, precio, estado, categoría (selecciona de las categorías raíz).
  • Toggle "Envío disponible".
  • Sección opcional "Lugar de recogida": descripción y coordenadas (longitud, latitud).

Dos estados independientes

const [datos, setDatos] = useState({
  titulo: '', descripcion: '', precio: '',
  estado: 'NUEVO', idCategoria: '', disponibilidadEnvio: false,
});
const [recogida, setRecogida] = useState({
  descripcion: '', longitud: '', latitud: '',
});

El submit hace dos llamadas API

const { id: idCreado } = await productosApi.crearProducto({
  ...datos,
  precio: Number(datos.precio),
  idVendedor: usuario.id,
});

if (idCreado && recogida.descripcion.trim()) {
  await productosApi.asignarRecogida(idCreado, {
    descripcion: recogida.descripcion.trim(),
    longitud: Number(recogida.longitud) || 0,
    latitud: Number(recogida.latitud) || 0,
  });
}

navegar(idCreado ? `/productos/${idCreado}` : '/perfil/productos');

Nótese cómo se extrae { id: idCreado } del resultado de crearProducto: la función está marcada con returnLocation: true en productos.js, así que devuelve { datos, location, id }.

AdminTransacciones — src/pages/AdminTransacciones.jsx

Qué hace

Tabla paginada de TODAS las compraventas con dos filtros opcionales: ID Comprador y ID Vendedor.

Truco del estado: dos copias

const [idComprador, setIdComprador] = useState('');     // input "borrador"
const [idVendedor, setIdVendedor] = useState('');       // input "borrador"
const [filtros, setFiltros] = useState({ idComprador: '', idVendedor: '' });  // aplicados

Esto evita lanzar peticiones al backend con cada tecla. Sólo al pulsar "Buscar" se copia el "borrador" a "filtros aplicados", y como ese cambio está en las dependencias del useEffect, ahí se dispara la petición.

const buscar = (e) => {

Para la entrevista

Si te piden "implementa una funcionalidad X", el guion es siempre el mismo: (1) ¿Qué página? (2) ¿Qué estado hace falta? (3) ¿Qué función de api/ se llama? (4) ¿Cuándo se llama: en useEffect (al montar/cambiar dep) o en respuesta a un evento (onClick/onSubmit)? (5) ¿Qué muestro mientras carga (Spinner), si hay error (Alert) y si no hay datos? (6) ¿Hay que proteger la ruta? Repasa una página cualquiera y verás que cumple este esqueleto.
